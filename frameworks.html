Core Engine

ICDF Engine

The foundational structure of every SIRVATO OS prompt. ICDF provides the four essential pillars that make any AI request precise and actionable.

I

Instruction

The action verb + specific task. What the AI must do.

C

Context

The scenario, background, and situational awareness the AI needs.

D

Data

All material, facts, content, and input the AI should use.

F

Format

The exact output specification — structure, length, style.

🟣

Role (R)

Define who the AI is. Expert, consultant, assistant — role shapes perspective and tone.

🔵

Context (C)

The full scenario: objective, audience, background, constraints.

🟡

Request (R)

The precise ask — specific, actionable, measurable.

🟪

Examples (E)

Reference models, counter-examples, and calibration samples.

🔵

Output (O)

Exact output specification: format, length, structure, style.

🟡

Constraints (C)

Hard limits, excluded topics, tone restrictions, style rules.

Validation Protocol

AI Trigger Protocol

The 5-trigger validation system that ensures every prompt is coherent, complete, and ready before execution. No prompt launches without passing all five checks.

1

Coherence

Role + Instruction are aligned and non-contradictory.

2

Rhythm

Tone and energy are explicitly defined.

3

Data

All required material and context is present.

4

Output

The desired output is unambiguous and specific.

5

Constraints

Limits, exclusions, and style rules are stated.
