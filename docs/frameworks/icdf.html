What is the ICDF Engine?

The ICDF Engine is the foundational cognitive layer of SIRVATO OS. It processes every incoming signal — user commands, system events, API calls — through a four-stage pipeline that transforms raw input into precise, contextually-aware decisions.

Unlike traditional rule-based engines, ICDF uses intent modeling to understand what you want, not just what you said. It maintains rich context across sessions and produces decisions that account for system state, historical patterns, and predicted outcomes.

Core Principle

Every action in SIRVATO OS flows through ICDF. From a simple health check to a complex multi-region failover — the same four-stage pipeline ensures consistency, traceability, and correctness.

The Four Stages

Stage 1
🎯
Intent

Natural language processing and semantic analysis to determine what the user or system truly intends — beyond literal commands.

Stage 2
📋
Context

Aggregates system state, history, environment variables, and operational constraints to build a complete situational picture.

Stage 3
⚖️
Decision

AI-powered decision engine evaluates options against constraints, risk models, and optimization objectives to select the optimal action.

Stage 4
🔄
Flow

Orchestrates execution across the system, routing to the appropriate framework, monitoring progress, and handling exceptions.
