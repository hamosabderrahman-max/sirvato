FRAMEWORK

ICDF Engine

Instruction-Context-Data-Format Structural Framework

The foundational structural layer of SIRVATO OS. Every prompt is decomposed into 4 precision layers for maximum clarity and AI comprehension.

Overview

What is the ICDF Engine?

The ICDF Engine is the core structural primitive of SIRVATO OS. It provides a repeatable, universal framework for decomposing any prompt into four distinct, precision-crafted layers. By separating Instruction from Context, and Data from Format, the ICDF Engine eliminates ambiguity at the source — the single greatest cause of poor AI output. Whether you are writing a one-line prompt or orchestrating a complex multi-step task, ICDF gives you a rigorous scaffold that AI systems understand and respond to with consistency.

Every other SIRVATO OS framework is built on top of the ICDF foundation. Mastering ICDF means mastering the grammar of prompt engineering itself.

Layer Architecture

The 4 Precision Layers

I
Instruction The precise action to perform. Be explicit. Verbs first. Ambiguous instructions produce ambiguous output — every word here must carry intent. Start with a strong action verb: write, analyze, generate, transform, extract. Define exactly what the AI must do, leaving no room for interpretation.
C
Context The scenario, environment, and background. AI needs context to calibrate its response. Without context, the model defaults to generic. With rich context, it specializes. Include the domain, the audience, the situation, the constraints implied by the environment, and any background knowledge the AI needs to perform the instruction accurately.
D
Data The raw material, facts, examples, or content to work with. This is the input the AI processes. It can be a text to rewrite, a dataset to analyze, a list of facts to synthesize, reference examples for style matching, or any concrete material the instruction requires. If you have no data, state that explicitly so the AI knows it must generate or infer.
F
Format The exact output form: length, structure, style, tone. The Format layer is the final precision control. Specify whether you want a list, a table, a narrative, code, JSON, markdown. Set the word count or character limit. Define the heading structure. Set the tone — formal, casual, technical, persuasive. Format is what transforms a correct response into a usable one.
Code Example

ICDF in Practice

ICDF Structure — Investor Pitch 
ICDF STRUCTURE:
I — Instruction : Scrivi un pitch per investitori
C — Context     : Startup AI nel settore healthcare, round seed
D — Data        : Team di 3 ingegneri, MVP completato, 50 beta user
F — Format      : Email professionale, 200 parole max

─────────────────────────────────────────────────────
OUTPUT PROMPT (assembled):

Scrivi un pitch per investitori per una startup AI nel settore 
healthcare in fase seed. Il team è composto da 3 ingegneri, 
l'MVP è stato completato e si contano 50 beta user attivi.
Formato: email professionale, massimo 200 parole.
─────────────────────────────────────────────────────
How It Works

4-Step Execution Process

01

Decompose the Goal

Start by identifying what you want to achieve. Break that goal apart into its instruction component (what to do), its context component (what setting this occurs in), its data component (what material to work with), and its format component (what the output should look like). Don't write the prompt yet — just think in layers.

02

Write Each Layer With Precision

Fill in each layer independently. Write the Instruction as a crisp command. Write the Context as a scene-setting paragraph. Collect and organize the Data. Define the Format with specifics — not "a list" but "a numbered list of 5 items, each under 20 words". Precision at the layer level creates clarity at the output level.

03

Combine in ICDF Order

Assemble the four layers into a single coherent prompt in I → C → D → F sequence. This order matches the natural processing flow of most AI systems: understand the task, calibrate to context, process the data, produce in the specified format. Deviating from the order often reduces output quality.

04

Iterate the Format Layer First

When you need variations of the same output, change the F layer only and keep I, C, D stable. This lets you generate a blog post, a slide deck outline, a tweet thread, and an executive summary from a single ICDF structure with minimal effort. Format iteration is the fastest path to multi-format content production.

When to Use

Ideal Use Cases

✦

Single-Task Prompts Needing Structure

Any time you need a single, well-defined output from a standalone prompt, ICDF provides the structural clarity to get it right the first time. Reduces prompt iteration cycles by up to 70%.

✦

Documentation and Content Creation

ICDF excels at content tasks where the output format is as important as the content itself. Perfect for reports, articles, emails, proposals, and any written deliverable with defined specifications.

✦

Prompts Where Output Format Matters

When the shape of the output is a critical requirement — structured data, specific markdown, code in a particular language, tables with defined columns — the Format layer ensures precision delivery every time.

✦

Teaching AI a New Task From Scratch

When introducing the AI to a domain, process, or task it hasn't encountered in your session, ICDF's layered structure ensures the model has everything it needs to understand the job before attempting it.

✦

Foundation for Other SIRVATO Frameworks

ICDF is the base layer beneath MICRO, RCR-EOC, AUTO-FLOW, and Unified Mode. Understanding ICDF deeply is the prerequisite for using all other SIRVATO OS frameworks at full effectiveness.
