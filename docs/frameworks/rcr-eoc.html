RCR-EOC

Role · Context · Request · Examples · Output · Constraints — complete scenario framework for complex tasks.

Components

  • R — Role The persona, expertise level or character the AI should embody.
  • C — Context Full situation: project background, stakeholders, history, environment.
  • R — Request The precise task, action or deliverable being requested.
  • E — Examples Models to follow, counter-examples to avoid, references for style/quality.
  • O — Output The specific format, length, structure and delivery method.
  • C — Constraints Hard limits: word count, tone, topics to avoid, must-include elements.

Example

🟪 RCR-EOC FRAMEWORK
────────────────────────────────────────
Role:        Senior UX researcher with 10y experience
Context:     SaaS onboarding redesign, B2B product, 500 users
Request:     Write 5 user interview questions for pain point discovery
Examples:    Open-ended questions, avoid yes/no, JTBD framework
Output:      Numbered list with brief rationale per question
Constraints: Max 15 words per question, no leading questions
────────────────────────────────────────
→ Ready to execute

Best Use Cases

  • Research tasks
  • UX/product work
  • Complex writing
  • Strategic planning
