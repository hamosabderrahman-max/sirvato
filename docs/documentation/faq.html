Frequently Asked Questions

Answers to the most common SIRVATO OS questions

What is SIRVATO OS?

SIRVATO OS is a prompt engineering operating system — a structured collection of 8 frameworks, 8 interfaces, and the MICRO SUPRA-TOTALITY architecture. It provides a systematic, repeatable methodology for crafting high-precision prompts for any AI system, from simple content tasks to complex enterprise pipelines.

Which framework should I start with?

Start with ICDF Engine if your task has clear structure (summarize, analyze, write). Use MICRO Layer when tone and communication style matter. If you are unsure, use the Framework Selector Tool to get a recommendation based on your use case.

Do I need to use all components of a framework?

No. Start with the components most relevant to your task. For ICDF, at minimum you need I (Instruction) and F (Format). Adding C and D improves quality but is not always required. For MICRO, M (Message) and O (Output) are the minimum viable pair. The more components you fill, the more precise the output.

What is the MICRO SUPRA-TOTALITY system?

MICRO SUPRA-TOTALITY is the apex architecture of SIRVATO OS. It activates all 5 MICRO layers simultaneously across 5 operating modes: Core, Reality+ (ambient adaptation), GODFIELD (absolute field), Terminal (shell interface), and Dashboard (widget visualization). It is designed for maximum-complexity prompt engineering tasks where every dimension of communication must be precisely controlled.

What is the AI Trigger Protocol?

The AI Trigger Protocol is a 5-point validation gate applied before sending any prompt. The 5 triggers are: T1 Coherence (components aligned), T2 Rhythm (tone defined), T3 Data (all information present), T4 Output (format unambiguous), T5 Constraints (limits confirmed). All 5 must pass before a prompt is considered ready.

Can I combine multiple frameworks?

Yes. Framework chaining is an advanced technique. A common pattern is using ICDF to structure content, then passing its output through MICRO to calibrate tone. The Unified Mode (11 components) is the built-in way to use all frameworks simultaneously in a single prompt.

Is SIRVATO OS specific to one AI system?

No. SIRVATO OS frameworks are AI-agnostic. They work with any large language model including GPT-4, Claude, Gemini, Mistral, and others. The structured approach improves output quality regardless of the underlying model.

How do I use the Live Demo?

Open the Live Demo page and fill in the 5 MICRO fields: Message, Intention, Context, Rhythm, and Output. The prompt preview updates in real time. Press Ctrl+Enter to generate the final structured prompt, then copy it to your AI system with the Copy button.

What does AUTO-FLOW mean?

AUTO-FLOW is a family of three frameworks for multi-step prompt execution. AUTO-FLOW Sequential runs steps in a fixed order. AUTO-FLOW PRO provides guided, interactive step-by-step progression. AUTO-FLOW REACTIVE adapts its flow based on the output of each previous step, making it ideal for dynamic or unpredictable workflows.
