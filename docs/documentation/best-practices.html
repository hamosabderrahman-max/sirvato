Best Practices

Expert techniques to get maximum precision from every prompt

Core Principles

🎯

Specificity Over Brevity

Detailed, specific prompts consistently outperform short vague ones. Define every component clearly — the AI cannot infer what you did not write.

🏗️

Structure First

Always apply a framework before writing prose. Structure forces you to identify gaps before they reach the AI.

🔄

Iterate Systematically

Change one component at a time when iterating. This makes improvements reproducible and traceable.

📋

Template and Reuse

Successful prompts are templates. Save them, version them, and share them across your team for consistent results.

Framework Selection Guide

When To Use Each Framework 
USE ICDF WHEN:
  ✓ Task is clearly structured (summarize, translate, classify)
  ✓ Output format is well-defined (table, list, report)
  ✓ Data is the primary driver of the output

USE MICRO WHEN:
  ✓ Tone, voice, or emotional impact matters
  ✓ Communication is the end goal (email, pitch, speech)
  ✓ Rhythm and energy need to be calibrated

USE RCR-EOC WHEN:
  ✓ Role-playing or persona is needed
  ✓ Professional or business context
  ✓ Hard constraints and expected outputs are critical

USE AUTO-FLOW WHEN:
  ✓ Multi-step process needs sequential execution
  ✓ Each step depends on the previous output
  ✓ Pipeline automation is the goal

USE UNIFIED MODE WHEN:
  ✓ High-stakes deliverables require maximum coverage
  ✓ Multiple teams or stakeholders involved
  ✓ Completeness is more important than speed

Component Writing Tips

01

Write Instructions as Actions

Use imperative verbs. Be specific about the action.

Instruction Examples 
❌ WEAK:    "Something about quarterly results"
✓ STRONG:  "Write a 300-word executive summary of Q3 revenue performance"

❌ WEAK:    "Code stuff"
✓ STRONG:  "Refactor this Python function to use list comprehension and add type hints"
02

Context Should Eliminate Ambiguity

Context tells the AI what world it's operating in. Include audience, environment, and constraints.

Context Examples 
❌ WEAK:    "Business context"
✓ STRONG:  "B2B SaaS startup, 50 employees, targeting enterprise finance teams, 
            Series A funding round, audience = technical CFOs"
03

Format Constrains, It Doesn't Suggest

Don't say "in a nice format". Say exactly what you need.

Format Examples 
❌ WEAK:    "Nice format"
✓ STRONG:  "Markdown table with columns: Feature | Benefit | Priority (H/M/L). 
            Max 8 rows. No introductory text."
04

Use The AI Trigger Protocol as a Final Gate

Before sending any prompt, run the 5-trigger check mentally or in writing.

Pre-send Checklist 
PRE-SEND VALIDATION:
[ ] T1 — Coherence  : All components aligned and non-contradictory
[ ] T2 — Rhythm     : Tone defined and consistent throughout
[ ] T3 — Data       : All referenced data is included or clearly referenced
[ ] T4 — Output     : Desired format is precise and unambiguous
[ ] T5 — Constraints: Exclusions, limits, and rules are explicit

→ All 5 checked? SEND. Otherwise, revise.

Common Anti-Patterns

What To Avoid 
ANTI-PATTERN 1 — THE VAGUE BRIEF
  ❌ "Make it better"
  ✓  "Improve clarity: shorten sentences to under 20 words and replace 
      passive voice with active voice"

ANTI-PATTERN 2 — MISSING FORMAT
  ❌ "Explain quantum computing"
  ✓  "Explain quantum computing in 5 bullet points, each under 20 words, 
      targeting a business executive with no technical background"

ANTI-PATTERN 3 — CONTEXT-FREE DATA
  ❌ [paste raw data with no explanation]
  ✓  "The following CSV represents monthly sales by region. 
      Focus only on Q4 2023. [data]"

ANTI-PATTERN 4 — CONTRADICTORY CONSTRAINTS
  ❌ "Write a 500-word tweet"  (impossible)
  ✓  "Write a tweet thread of 5 tweets, each under 280 characters"

ANTI-PATTERN 5 — ROLE MISMATCH
  ❌ "As a junior developer, write an enterprise architecture document"
  ✓  "As a senior solutions architect with 10 years enterprise experience, 
      write an architecture decision record"

Advanced Techniques

🔗

Chain Frameworks

Use ICDF to structure content, then pass its output through MICRO to calibrate tone. Each framework adds a precision layer.

📖

Include Negative Examples

Show the AI what NOT to produce. Negative examples reduce the solution space and prevent common failure modes.

⚡

Progressive Activation

Start with the minimal framework and add components when quality is insufficient. Don't over-engineer from the start.

🎭

Role Precision

The more specific the role, the better the output. "Senior product manager at a fintech startup with 8 years experience" outperforms "product manager".
