Best Practices

Tips, patterns and advanced techniques from real-world SIRVATO OS usage.

📖 Documentation

The Golden Rules

  • Be SpecificVague prompts get vague results. Every field should add precision, not approximation.
  • Define Output FirstKnow exactly what you want to receive before filling in the other fields. Work backwards.
  • Rhythm MattersTone, energy and pacing dramatically affect AI output quality. Never skip the RHYTHM field.
  • Constraints WinWhat to NOT do is as powerful as what to do. Always include explicit exclusions.
  • Examples AnchorProviding concrete examples reduces interpretation variance by 60-70%.

Advanced Techniques

Layered Prompting

Start with HUD Minimal to explore, then graduate to ICDF for structure, then Unified Mode for final production output.

Trigger Validation

Before running Unified Mode, mentally check all 5 AI Triggers. If any fail, refine that section first.

Rhythm as Multiplier

The MICRO Rhythm field acts as a multiplier — even a simple "energetic and punchy" doubles output quality for content tasks.

Common Mistake: Filling all 11 fields of Unified Mode with generic, vague text. It's better to use ICDF with 4 precise fields than Unified Mode with 11 fuzzy ones.

Framework Stacking

Advanced users stack frameworks: first use RCR-EOC to validate the scenario, then feed the output into Auto-Flow PRO for refinement, then validate with AI Trigger Protocol before executing Unified Mode.

Stack: RCR-EOC → Auto-Flow PRO → AI Triggers → Unified → Execute
