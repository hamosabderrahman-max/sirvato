Getting Started with SIRVATO OS

This guide walks you through installing SIRVATO OS, initializing your first environment, and deploying your first autonomous workload. The entire process takes under 10 minutes.

1

Installation

Install the SIRVATO CLI using npm or the official install script.

BASH
# via npm
npm install -g @sirvato/cli

# or via install script
curl -fsSL https://install.sirvato.io | bash

# verify installation
sirvato --version
# SIRVATO OS CLI v3.1.0
2

Initialize Your Environment

Initialize a new SIRVATO project in AUTOPILOT mode — the recommended mode for all new deployments.

BASH
sirvato init --mode=autopilot

# Output:
# ✓ Project initialized
# ✓ sirvato.yaml created
# ✓ AUTOPILOT mode enabled
# ✓ AI framework selected: ICDF
3

Configure sirvato.yaml

Edit the generated configuration file to match your infrastructure requirements.

YAML
# sirvato.yaml
project:
  name: "my-app"
  mode: "autopilot"

framework: "icdf"

deploy:
  region: "us-east-1"
  replicas: 3
  auto_scale: true

security:
  zero_trust: true
  encrypt_at_rest: true
4

Deploy

Run the deploy command. SIRVATO OS handles provisioning, configuration, and health checks automatically.

BASH
sirvato deploy

# ✓ Provisioning infrastructure...
# ✓ Configuring networking...
# ✓ Deploying application...
# ✓ Running health checks...
# ✓ Deployment complete in 47s
# → https://my-app.sirvato.cloud
5

Monitor

Check the live status of your deployment including metrics, health, and AI decisions.

BASH
sirvato status

# Project:    my-app
# Status:     HEALTHY
# Uptime:     99.99%
# Replicas:   3/3 running
# Latency:    4.2ms p99
# AUTOPILOT:  ACTIVE
6

Enable Auto-Scale

Activate autonomous scaling so SIRVATO OS manages capacity without any manual intervention.

BASH
sirvato scale --auto

# ✓ Auto-scaling enabled
# ✓ Min replicas: 2
# ✓ Max replicas: 50
# ✓ Scale trigger: CPU > 70%
# ✓ AUTOPILOT managing scale
