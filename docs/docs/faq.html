FAQ

Frequently asked questions about SIRVATO OS, its frameworks, and best usage patterns.

📖 Documentation

General Questions

What is SIRVATO OS?

SIRVATO OS is a unified prompt engineering operating system that combines multiple frameworks (ICDF, MICRO, RCR-EOC, Auto-Flow) into a single, coherent system for crafting powerful AI prompts.

Which AI models does it work with?

SIRVATO OS is model-agnostic — it works with any language model: GPT-4, Claude, Gemini, Llama, Mistral, and others. The framework improves prompt quality regardless of the underlying model.

Do I need to use all frameworks?

No. Start with the simplest framework that meets your needs. HUD Minimal and ICDF are great entry points. Graduate to Unified Mode when maximum precision is required.

Framework Questions

What is the difference between ICDF and MICRO?

ICDF focuses on structure (Instruction → Context → Data → Format) and is ideal for precise, technical tasks. MICRO adds rhythm and emotional resonance (Message → Intention → Context → Rhythm → Output) making it ideal for content creation.

When should I use Unified Mode?

Use Unified Mode when the stakes are high, the task is complex, or you need maximum precision. Examples: product launches, enterprise documentation, high-conversion copy, strategic plans.

What are the AI Triggers?

The AI Trigger Protocol is a 5-point validation checklist that ensures your prompt is complete before execution: Coherence, Rhythm, Data, Output, and Constraints must all pass before activating Unified Mode.

What does "Reactive" mean in Auto-Flow Reactive?

In Reactive mode, each field you fill automatically informs and activates the next field. It's an adaptive cascade where the system responds to your input in real-time, creating a dynamic prompt construction experience.

Technical Questions

Is SIRVATO OS open source?

Yes. SIRVATO OS is available on GitHub and is open source.

How do I contribute?

Fork the repository, create a branch, make your changes, and open a pull request. Contributions to documentation, framework definitions, and interface improvements are all welcome.

Where do I report issues?

Open an issue on the GitHub Issues page.
