MICRO AUTO-TRIGGER

Self-Activating Cascade Prompt System

AUTO-TRIGGER is the autonomous activation layer. Once initialised, it self-monitors prompt quality, automatically triggers corrections, amplifications, and adaptations without manual intervention. The system improves itself in real time.

Overview

What is MICRO AUTO-TRIGGER?

MICRO AUTO-TRIGGER is the fifth and penultimate layer of the MICRO SUPRA stack, sitting directly below SUPRA-TOTALITY. It is the automation layer — the system that removes manual quality management from the prompt engineering workflow by replacing human monitoring with a set of intelligent, self-activating triggers. Once you activate AUTO-TRIGGER at the start of a session, it runs continuously in the background, monitoring every prompt you build against a set of quality thresholds and firing correction sequences whenever a threshold is breached.

The cascade principle is central to AUTO-TRIGGER's design. Triggers do not operate in isolation — they form a cascade. When the INIT TRIGGER fires at session start, it initialises all other triggers. When the QUALITY TRIGGER fires, it may cascade into the CONTEXT TRIGGER if the quality failure is context-related. When the MASTER TRIGGER fires — the highest-severity trigger — it cascades through every other trigger in the system, performing a complete reset and recalibration. This cascade architecture ensures that every quality problem is addressed at the correct level of the system, with minimum disruption and maximum precision.

Trigger Cascade Sequence

The 7 AUTO-TRIGGERs

T0
INIT TRIGGER — Session Self-Initialisation The INIT TRIGGER fires automatically when you begin typing the first MICRO component of a new session. It reads the SUPRA-OS kernel state, loads persisted session context from MEMORY, activates the HOLOFIELD layers, and arms all subsequent triggers. The INIT TRIGGER also performs a compatibility check — verifying that the active frameworks, interface mode, and AI model driver are all compatible before allowing any other triggers to arm. A failed INIT requires manual resolution before the session proceeds.
T1
QUALITY TRIGGER — Output Quality Monitoring The QUALITY TRIGGER runs a continuous quality assessment against the assembled MICRO prompt, comparing it against the quality model for the active task profile. When the quality score drops below the configured threshold (default: 75/100), the QUALITY TRIGGER fires. The automatic response is a staged intervention: first, a non-blocking notification with specific improvement suggestions. If the quality score does not improve within 3 editing cycles, the QUALITY TRIGGER cascades into an AUTO-ADAPT sequence that applies the highest-scoring improvement suggestion automatically.
T2
CONTEXT TRIGGER — Context Drift Detection The CONTEXT TRIGGER monitors the C (Context) component for drift — the gradual accumulation of inconsistencies between the stated context and the other MICRO components. Context drift often occurs in long sessions where the Message or Intention components evolve but the Context is not updated to reflect the evolution. When drift is detected, the CONTEXT TRIGGER fires and presents a Context Sync panel showing the specific inconsistencies and offering targeted context update suggestions.
T3
RHYTHM TRIGGER — Tone Deviation Correction The RHYTHM TRIGGER monitors the R (Rhythm) component and the overall tonal consistency of the assembled prompt. It detects tone deviation — when the rhythm specification conflicts with the language used in other components. For example: if R specifies "formal, measured" but the Message uses casual contractions, the RHYTHM TRIGGER fires. The automatic correction options range from adjusting the Message language to match the Rhythm, to flagging the Rhythm specification as potentially misaligned with the task intent and suggesting a recalibration.
T4
DATA TRIGGER — Information Gap Detection The DATA TRIGGER analyses the MICRO structure for information gaps — places where the assembled prompt requires data that has not been provided. When an ICDF prompt references a D (Data) field that is empty, or when an MICRO Context references an audience characteristic without defining it, the DATA TRIGGER fires. The automatic response is a Data Request Panel that lists each detected gap with a specific, targeted question for the user to answer, plus an option to search the FILESYSTEM for relevant saved content that might fill the gap.
T5
OUTPUT TRIGGER — Delivery Specification Enforcement The OUTPUT TRIGGER enforces the O (Output) specification. When a generated AI response does not match the Output specification — wrong length, wrong format, wrong structure — the OUTPUT TRIGGER fires. It analyses the deviation: is the AI response too long? Structurally incorrect? Missing required sections? The AUTO-TRIGGER response is a reformatting instruction sent back to the AI as a follow-up prompt, automatically requesting a corrected output that meets the full Output specification. The user receives the corrected output, not the original deviation.
T6
MASTER TRIGGER — Full System Reset and Recalibration The MASTER TRIGGER is the emergency override — fired when multiple triggers have failed to resolve a quality problem through their normal cascade sequences. When the MASTER TRIGGER fires, it performs a complete system reset: all MICRO components are archived to the FILESYSTEM, the SUPRA-OS kernel is cleared, and the session is reinitialised from scratch using the archived components as starting reference material. The user is presented with a structured recalibration guide — a step-by-step process for rebuilding the MICRO structure correctly from the ground up, with the original components available for reference.

Key Capabilities

AUTO-TRIGGER in Practice

✦

Zero-Manual-Monitoring Workflow

With AUTO-TRIGGER active, you never need to manually check your prompts for quality issues. The 7-trigger cascade watches every component continuously, compares against quality thresholds, and either fixes problems automatically or surfaces targeted, specific correction requests. The net result is a workflow where prompt quality management is fully automated — you focus on content, AUTO-TRIGGER manages quality.

✦

Configurable Trigger Thresholds

Every trigger has configurable thresholds. The QUALITY TRIGGER default of 75/100 can be raised to 90/100 for maximum-precision work or lowered to 60/100 for fast-iteration drafting sessions where speed matters more than perfection. The CONTEXT TRIGGER's drift sensitivity can be adjusted. The DATA TRIGGER can be set to block generation until all gaps are filled, or to allow generation with gaps and surface a post-generation gap report instead.

✦

Cascade Transparency Log

Every trigger firing, every cascade, and every automatic correction is logged in the AUTO-TRIGGER Cascade Log — a timestamped event stream accessible in the SUPRA-OS dashboard. The log shows exactly which trigger fired, what condition caused it, what automatic response was applied, and the before/after quality scores. This transparency turns the automated quality management system into a learning resource — you can study your own prompt patterns to understand where your weaknesses consistently appear.

✦

Integration with AI Trigger Protocol

AUTO-TRIGGER is distinct from but integrates with the AI Trigger Protocol framework. The AI Trigger Protocol is a pre-generation validation framework — it runs 5 checks before the prompt is sent to the AI. AUTO-TRIGGER is a continuous monitoring system that runs throughout the session. Together, they provide complete coverage: AUTO-TRIGGER monitors the prompt as it is built; the AI Trigger Protocol validates it before it is sent. No quality gap between session start and generation completion.
