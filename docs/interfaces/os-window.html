OS Window

Classic OS Window Multi-Pane Interface

A familiar desktop OS-style window system. Multiple resizable panes for simultaneous framework display, output preview, and context management — all in a traditional windowed UI paradigm.

Overview

What is OS Window?

OS Window bridges the familiar world of desktop software with the precision of SIRVATO OS prompt engineering. Users coming from traditional IDEs, document editors, or any application with a classic windowed interface will recognise the layout immediately: a menu bar at the top, a sidebar on the left, a main working area in the centre, and a status bar along the bottom. Everything works exactly as expected — and adds the full power of the SIRVATO OS framework stack on top.

The pane system is fully draggable and resizable. You can dedicate the entire window to the prompt editor, or split the space to view your live output preview alongside your active framework structure. Tabbed history lets you maintain multiple in-progress prompts and switch between them without losing any work. For users who need a comfortable, low-surprise environment to learn SIRVATO OS, OS Window is the recommended starting point.

Features

Core Capabilities

✦

Draggable and Resizable Panes

Every pane boundary is a drag target. Drag the divider between the framework sidebar and the prompt editor to give yourself more editing space. Drag the horizontal divider to reveal or hide the output preview. All pane sizes are saved to local storage and restored on next session open.

✦

Framework Selector Sidebar

The left sidebar lists all eight SIRVATO OS frameworks with a single-click to activate. The active framework is highlighted and its component labels appear in the prompt editor area. Hover over any framework name for a tooltip summary. Click the pin icon to lock a framework for the session.

✦

Live Output Preview Panel

A collapsible bottom panel shows a live formatted preview of your assembled prompt as you type. The preview renders the framework structure with labels and separators, giving you a WYSIWYG view of what the AI will receive. Toggle the preview with Ctrl+P or use View → Preview from the menu bar.

✦

Menu Bar with Keyboard Shortcuts

A full menu bar mirrors the conventions of desktop software: File (new, open, save, export), Edit (undo, redo, copy, paste, select all), View (toggle panels, zoom), Framework (switch, configure), Tools (validate, generate, batch), and Help (docs, shortcuts). Every menu item displays its keyboard shortcut.

✦

Status Bar with System Info

The persistent status bar at the bottom displays the active framework, current token count, system ready state, last save time, and cursor position. The rightmost segment shows the SIRVATO OS version and a quick-access icon for settings. Double-click any status segment for detailed information about that metric.

✦

Tabbed Prompt History

Open prompts appear as tabs across the top of the editor area, just like browser tabs or IDE editor tabs. Ctrl+T opens a new prompt tab. Ctrl+W closes the current tab. Ctrl+Shift+T reopens the last closed tab. Each tab retains its framework state, content, and output independently.

✦

File Save and Load for Prompt Templates

Save any prompt as a named template with File → Save Template. Templates are stored locally and appear in a template picker accessible from File → Open Template. Templates preserve the full SIRVATO OS structure — framework selection, all field values, and output configuration — ready to reuse or share with your team.
