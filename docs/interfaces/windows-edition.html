Windows Edition

Multi-Panel Windows-Inspired Interface System

A comprehensive multi-window desktop environment for power users who need to manage multiple prompt workflows simultaneously, with full OS-level snap, tile, and window management integration.

Overview

What is Windows Edition?

Windows Edition is SIRVATO OS's answer to the professional multi-monitor, multi-workflow power user. Where OS Window gives you a single structured environment, Windows Edition opens the entire desktop as your working surface. Spin up as many independent prompt windows as you need. Snap two windows side by side for A/B comparison. Tile three windows to run MICRO, ICDF, and Unified Mode outputs against the same underlying task simultaneously. Each window is a complete, independent SIRVATO OS session.

The Windows Edition is particularly powerful for enterprise prompt engineering workflows where standardisation matters. Windows can share a Context object — update the shared context once and every open window receives the updated information automatically. Cross-window prompt comparison renders outputs side by side with diff highlighting so you can see exactly where different framework choices produce different results. For teams deploying SIRVATO OS at scale, Windows Edition is the production-grade interface of choice.

Features

Core Capabilities

✦

Multiple Simultaneous Prompt Windows

Open any number of independent SIRVATO OS windows. Each window runs its own framework, maintains its own history, and generates its own output. Windows can be minimised, maximised, resized, or moved freely across the workspace. There is no limit to the number of concurrent windows, though system resources apply.

✦

Window Snap and Tile Functionality

Drag any window to a screen edge to snap it into a half-screen or quarter-screen tile. The snap system follows conventions established by modern desktop operating systems — drag to left edge for 50% left, top-right corner for 25% top-right, and so on. Snap layouts can be saved as named configurations and recalled with a single click or keyboard shortcut.

✦

Shared Context Between Windows

Create a Shared Context object and link multiple windows to it. When you update the shared context — changing a company name, switching the target audience, updating a data set — every linked window instantly receives the update and can regenerate its output against the new context. This eliminates copy-paste errors when running parallel framework comparisons.

✦

Cross-Window Prompt Comparison

Select two or more windows and activate Compare Mode. The interface renders all selected window outputs side by side with colour-coded diff highlighting showing exactly where the outputs differ. Differences are categorised by type — structural differences (blue), content differences (amber), tone differences (purple) — giving you a nuanced analysis of how framework choice affects output quality.

✦

Task Switcher (Alt+Tab Style)

Alt+Tab cycles through all open SIRVATO OS windows exactly as you would expect in a desktop OS. The task switcher shows a thumbnail preview of each window's current content, the active framework, and the last activity timestamp. Click any thumbnail to bring that window to focus immediately, or continue pressing Tab to cycle through all open windows.

✦

System Tray with Quick Actions

A persistent system tray in the bottom-right corner provides quick access to the most common actions: new window, copy active output, switch framework, open block library, and session save. The tray is always visible regardless of which window has focus, making global actions accessible without navigating away from your current work.

✦

Window State Persistence

Every aspect of your Windows Edition session is saved automatically — window positions, sizes, snap configurations, active frameworks, prompt content, and output history. Close and reopen the application and your complete workspace is restored exactly as you left it. Session snapshots can be exported, shared with teammates, and loaded on a different machine.

Usage Example

Windows Edition in Action

Open Windows Edition and use the New Window button (Ctrl+N) or the system tray to spin up additional windows. Snap the first window left (MICRO framework), the second window right (ICDF framework), and activate Shared Context to link both. Type your task once in the shared context and both windows generate their framework-specific outputs simultaneously:
