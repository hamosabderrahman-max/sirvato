Fullscreen Mode

Immersive Full-Viewport Prompt Environment

Total viewport takeover for maximum focus. Every pixel serves the prompt. Designed for deep work sessions where distraction is the enemy and the quality of your thinking is the only variable that matters.

Overview

What is Fullscreen Mode?

Fullscreen Mode is the interface that removes the interface. When you activate Fullscreen Mode — press F11 or the fullscreen button — the browser chrome disappears, the navigation disappears, the status bar disappears. What remains is your prompt, your framework, and the blank expanse of the viewport. Nothing else competes for your attention. The entire screen is dedicated to the single prompt you are crafting.

Behind the input area, a slow-moving ambient matrix animation provides subtle visual depth without inducing distraction — the motion is peripheral and calm, registering as texture rather than movement. The effect is closer to sitting in front of a starfield than watching a video. Users report that Fullscreen Mode induces a focused mental state similar to the distraction-free writing modes found in premium text editors. For long-form prompt crafting, complex multi-component structured prompts, or extended strategy sessions, Fullscreen Mode is the environment of choice.

Features

Core Capabilities

✦

True Fullscreen Activation

Press F11 to enter true browser fullscreen — the title bar, tabs, and all browser chrome are completely hidden. Alternatively, click the fullscreen icon in the top-right corner of the interface for an in-page fullscreen that maximises the SIRVATO OS workspace without triggering browser fullscreen. Both modes eliminate visual distractions; true fullscreen eliminates OS chrome as well.

✦

Ambient Matrix Background Animation

A subtle, low-opacity matrix animation plays behind the prompt area in Fullscreen Mode. The animation uses characters from the SIRVATO OS symbol set at 15% opacity and animates at 0.5x the speed of conventional matrix effects. The result is a living background that enhances the sense of working within a system, without pulling attention away from your prompt content.

✦

Large, Readable Input Areas

In Fullscreen Mode, input fields expand to their maximum comfortable reading size — 18px base font, 1.8 line height, with generous field padding. The text is large enough to be read at arm's length on a large monitor. This size also slows the pace of reading, which research consistently shows correlates with higher quality writing and more considered prompt construction.

✦

Minimal UI Chrome

The only visible chrome elements in Fullscreen Mode are the active framework label (top-centre, small and dim), the keyboard shortcut legend (bottom-centre, visible on hover), and the focus timer (top-right, optional). The framework switcher, token counter, and status LED are all hidden by default and appear only when you hover near the edges of the screen.

✦

Keyboard-Only Navigation

Fullscreen Mode is designed for a zero-mouse workflow. ⌘G generates the prompt, ⌘C copies output, ⌘F cycles frameworks, ⌘S saves the session, ⌘Z undoes, and Escape exits fullscreen. Tab moves between framework component fields in the correct logical order. The entire experience is navigable without lifting your hands from the keyboard.

✦

Auto-Save Every 30 Seconds

Fullscreen Mode auto-saves your entire prompt state every 30 seconds to local storage. A brief, non-intrusive save indicator (a single dot that fades in and out over 1 second) appears in the top-right corner on each save cycle. If your browser closes unexpectedly, the recovery system restores your last saved state on next open — with a timestamp showing exactly when the last save occurred.

✦

Focus Timer with Soft Alerts

An optional Pomodoro-style focus timer counts down from a configurable duration (default 25 minutes). When the timer expires, a soft ambient sound plays and the background animation briefly brightens to signal the break. The alert is designed to be noticeable without being startling — you finish your current thought before responding to the timer, not the other way around.
