Master Block

Structured Block-Based Prompt Organisation

An organised block system where each prompt component occupies a dedicated visual block. Build, reuse, and combine prompt modules with drag-and-drop simplicity — then assemble them into a Master Prompt Output.

Overview

What is Master Block?

Master Block reimagines prompt engineering as a visual block composition exercise, inspired by modular synthesis and block-based coding environments. Rather than writing a prompt as a single text stream, you build it as a collection of typed, labelled blocks — each block holding exactly one component of your chosen framework. Role in one block. Context in another. Request in a third. The interface then assembles the blocks into a complete, correctly formatted prompt in the Master Prompt Output zone.

The power of the block model becomes clear when you start building a block library. Save your best Role blocks, your standard Context blocks for recurring projects, your most effective Format specifications. Reuse them across dozens of prompts by dragging them in from the library. For teams, Master Block enables shared block libraries where prompt components become reusable IP — the same precision-crafted Role block can power hundreds of different prompts across the organisation.

Features

Core Capabilities

✦

Drag-and-Drop Block Reordering

Grab any block and drag it to a new position in the composition grid. The Master Prompt Output updates instantly to reflect the new block order. Reordering blocks lets you experiment with the sequence of framework components — sometimes placing Context before Role produces measurably better outputs for particular task types.

✦

Block Library for Saved Components

The Block Library panel sits alongside your composition area and holds all your saved blocks organised by type. Search the library by keyword, filter by framework type or block category. Drag any library block into your active composition. Blocks can be marked as favourites, tagged, and organised into named collections for different project types.

✦

Visual Block Connection Lines

Enable Connection Mode to see visual lines showing how blocks relate to each other within the active framework. Context connects to Instruction. Role connects to Request. The connection lines clarify the logical dependencies between components, helping new users understand framework structure and helping experienced users identify missing connections in complex compositions.

✦

Mix-and-Match from Different Frameworks

Master Block allows you to combine blocks from different SIRVATO OS frameworks in a single composition. Use an ICDF Instruction block, a MICRO Rhythm block, and an RCR-EOC Constraints block together. The Assemble engine validates cross-framework combinations and warns when components are incompatible, while permitting powerful hybrid structures that standard single-framework use doesn't allow.

✦

Export as JSON or Plain Text

Export the complete block composition — including all block content, types, library references, and framework metadata — as a structured JSON file. This JSON format is readable by SIRVATO OS tools, compatible with API integrations, and version-controllable in git. Alternatively, export as clean plain text for direct pasting into any AI interface.

✦

Block Templates for Common Patterns

The Templates gallery provides ready-to-use block compositions for the most common prompt engineering scenarios: content creation, code review, strategic analysis, customer communication, data transformation, and more. Each template is a complete block set ready to customise. Templates can be cloned, modified, and saved as new library entries.

✦

Colour-Coded Blocks by Framework Type

Each SIRVATO OS framework has a unique colour signature applied to all its blocks. ICDF blocks use deep blue, MICRO blocks use violet, RCR-EOC blocks use teal, AUTO-FLOW blocks use amber. At a glance you can see the framework composition of any prompt — whether it is a pure single-framework composition or a hybrid mix of components from across the SIRVATO OS stack.

Usage Example

Master Block in Action

Begin by selecting a framework from the top toolbar. The composition grid fills with empty blocks labelled with that framework's components. Fill each block, then click Assemble to generate the Master Prompt Output. The example below shows a four-block RCR-EOC composition and its assembled output:

Master Block — RCR-EOC Composition → Assembled Output 
BLOCK [ROLE]:        Senior Product Manager at a B2B SaaS company
BLOCK [CONTEXT]:     12-month roadmap planning cycle, 3 engineering squads
BLOCK [REQUEST]:     Build a prioritised roadmap for Q1–Q4 with milestones
BLOCK [CONSTRAINTS]: Max 10 initiatives, must include customer retention focus
BLOCK [EXAMPLES]:    Use OKR format for framing each initiative
BLOCK [OUTPUT]:      Slide-ready deck structure, 10 slides, exec summary first

── ASSEMBLE ──────────────────────────────────────

You are a Senior Product Manager at a B2B SaaS company.
Context: 12-month planning cycle, 3 engineering squads available.
Request: Build a prioritised roadmap for Q1–Q4 with clear milestones.
Constraints: Maximum 10 initiatives; must include customer retention focus.
Examples: Frame each initiative in OKR format.
Output: Slide-ready deck structure, 10 slides, executive summary first.

Best for: Reusable prompt components, team workflows where prompt components are shared assets, template building for standardised output quality, and users who prefer a visual spatial approach to prompt composition.
