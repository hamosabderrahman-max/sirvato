What is Master Block?

Master Block is the apex control surface for SIRVATO OS — a unified command center housing 15 deeply integrated modules covering every dimension of your system. From compute health to cost tracking, security posture to deployment pipelines, everything lives in one coherent, role-aware interface.

Each module is live-updating, cross-referenced, and aware of every other. A spike in the Performance Analyzer triggers a highlight in the Alert Hub. A change in the Config Editor propagates visible status updates to the Deployment Manager. The entire 15-module system operates as a single, interconnected intelligence.

Total System Intelligence

Master Block doesn't just aggregate dashboards — it synthesizes them. Context flows between modules so you always understand not just what is happening, but why and what to do about it.

Core Features

Feature 01
🏗️
15 Modules

Every operational concern — compute, network, security, cost, users, audit — covered by a dedicated, deeply integrated module.

Feature 02
⚡
Real-Time Data

All 15 modules stream live data simultaneously. No stale dashboards — every number, status, and alert is current.

Feature 03
🛡️
Role-Based Views

Module visibility and controls adapt to your RBAC role. Operators see operations; finance sees costs; admins see everything.

Feature 04
🎛️
Customizable Layout

Reorder, resize, and group modules into custom layouts. Save multiple named workspaces and switch between them instantly.

Module Configuration

YAML 
# sirvato.yaml — Master Block Modules
interfaces:
  master_block:
    enabled: true
    layout: "master"
    modules:
      compute_monitor: true
      network_map: true
      security_center: true
      storage_dashboard: true
      ai_insights: true
      log_stream: true
      deployment_manager: true
      config_editor: true
      alert_hub: true
      performance_analyzer: true
      cost_tracker: true
      user_manager: true
      api_explorer: true
      audit_trail: true
      command_palette: true
    rbac:
      admin: "all"
      operator: "ops-modules"
      finance: "cost-modules"
