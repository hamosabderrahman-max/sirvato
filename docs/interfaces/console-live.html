Console Live

Real-Time Terminal-Style Prompt Interface

A fully featured terminal interface inspired by Unix consoles and hacker aesthetics. Type, execute, and see responses in a live stream format with full command history and auto-complete.

Overview

What is Console Live?

Console Live brings the power and familiarity of the Unix terminal to prompt engineering. Every interaction flows through a scrolling output stream, timestamped and colour-coded by message type. Framework commands follow CLI conventions — developers and power users feel at home immediately. There is no GUI chrome to learn, no mouse required, and no menus to navigate. Everything is a command.

The live streaming output simulation mirrors how modern AI APIs deliver tokens — you see the response build character by character, giving you the ability to interrupt, redirect, or build on partial outputs before they complete. Console Live is the interface of choice for developers integrating SIRVATO OS prompts into automated pipelines and for engineers who think in terminals.

Features

Core Capabilities

✦

Live Streaming Output Simulation

Output renders token by token as it arrives from the AI, exactly mirroring real API streaming. Watch your prompt response build in real time. At any point you can interrupt output with Ctrl+C, capture what has appeared so far, or redirect the generation with a follow-up command without waiting for completion.

✦

Command History Navigation

Arrow Up / Arrow Down cycles through your full session history, exactly like a Unix shell. The history persists across browser sessions using local storage. Ctrl+R activates reverse history search — type a fragment of any previous command and the console finds the most recent match instantly.

✦

Auto-Complete for Framework Commands

Tab completion works for all SIRVATO OS commands: framework names, interface switches, configuration flags, and built-in operators. Double-Tab shows all available completions. The auto-complete engine is aware of your current framework context and surfaces only the commands relevant to your active mode.

✦

Colour-Coded Output by Message Type

System messages appear in cyan, prompt input echoes in white, AI output renders in green, warnings in amber, and errors in red. The colour coding is consistent across all message types and applies to both interactive sessions and exported logs, making session review fast and scannable.

✦

Export Full Session as Text

At any time, run export session to generate a complete plain-text log of the entire session — every input, every output, every timestamp. Export formats include raw text, markdown, JSON (with metadata), and ANSI colour-preserved terminal format for embedding in documentation.

✦

Keyboard-First Workflow

Console Live is designed to be operated entirely without a mouse. Every action has a keyboard shortcut. Ctrl+L clears the screen, Ctrl+K kills the current line, Ctrl+A moves to line start, Ctrl+E to line end. Power users build muscle memory that makes prompt engineering as fast as coding in a terminal.

✦

Timestamp on Every Output Line

Each line in the output stream carries a precise timestamp in [HH:MM:SS] format. This allows you to measure response latency, trace the exact sequence of events in complex multi-step sessions, and synchronise Console Live output with external system logs when integrating with automated workflows.

Usage Example

Console Live in Action

Open Console Live and begin typing immediately. The use command loads a framework, run executes the active prompt, and set configures output parameters. The following session demonstrates a complete MICRO framework interaction:

Console Live — MICRO Framework Session 
[14:23:01] > System initialized. SIRVATO OS Console Live v1.0
[14:23:01] > Type 'help' for available commands.

sirvato@os:~$ use micro
[14:23:04] > Framework: MICRO loaded. M-I²-C-R-O active.

sirvato@os:~$ set M "Launch announcement for our new developer API"
sirvato@os:~$ set I2 "Excite developers, drive early adoption"
sirvato@os:~$ set C "Tech audience, Hacker News and dev Twitter"
sirvato@os:~$ set R "High energy, peer-to-peer tone, no corporate speak"
sirvato@os:~$ set O "Tweet thread, 5 tweets, max 280 chars each"

sirvato@os:~$ run
[14:23:47] > Generating... ████████░░ 80%
[14:23:49] > Output stream:

Tweet 1: We just opened public access to the Sirvato API...
█

Best for: Power users, developers integrating prompts into pipelines, debugging multi-step prompt chains, and batch processing sessions where command-line speed is essential.
