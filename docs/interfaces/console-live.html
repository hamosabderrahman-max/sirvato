Console Live

Real-time live console with streaming output and interactive command execution. The power-user interface for SIRVATO OS.

Streaming Output Command Execution Plugin Support

What is Console Live?

Console Live is the power-user command center for SIRVATO OS. It combines a full-featured interactive terminal with a live-streaming log viewer, command history, and intelligent autocompletion — all in one unified interface.

Unlike static consoles, Console Live maintains a persistent WebSocket connection to your cluster, delivering real-time log streams, event feeds, and system notifications as they happen. Commands execute directly against the SIRVATO runtime with immediate, visible feedback.

Power User Design

Console Live was designed for the engineers who live in their terminals. Every feature — from multi-stream splitting to inline grep filtering — is built around the assumption that you know exactly what you're doing.

Core Features

Feature 01
📡
Streaming Output

Persistent WebSocket connection delivers log lines, events, and alerts in real time with <50ms end-to-end latency.

Feature 02
⚙️
Command Execution

Full SIRVATO CLI access directly in the console. Authenticated sessions with role-aware command resolution.

Feature 03
🔍
Log Filtering

Inline grep, regex filters, level filters (INFO/WARN/ERROR), and multi-pod stream splitting built into the UI.

Feature 04
🔌
Plugin Support

12 built-in plugins: metrics sidebar, AI assistant, alert rules editor, query builder, and more. Community plugins via registry.

Configuration

YAML 
# sirvato.yaml — Console Live Interface
interfaces:
  console_live:
    enabled: true
    port: 7700
    stream:
      buffer_lines: 10000
      latency_target: "50ms"
      reconnect_interval: "2s"
    filters:
      default_level: "INFO"
      highlight_errors: true
    plugins:
      - "metrics-sidebar"
      - "ai-assistant"
      - "alert-rules"
    auth:
      require_token: true
      rbac_enabled: true

Stream Commands

sirvato logs stream --all

Stream all pod logs in real time

sirvato logs stream --filter ERROR

Filter to error-level logs only

sirvato exec --pod <name> -- <cmd>

Execute command in a running pod

sirvato console split --streams 3

Split view into multiple streams
