Quick Start Guide

Get SIRVATO OS running in three steps. The quickest path to a production-ready prompt is the FULLSCREEN TOTALITY interface.

Step 1: Choose Your Interface

Start with the interface that matches your experience level:

  • New to SIRVATO OS? → Start with HUD Minimal
  • Familiar with prompt engineering? → Use Console Live
  • Want the full power? → Launch FULLSCREEN TOTALITY

Step 2: Fill the Core Fields

At minimum, complete these four fields for a functional prompt:

⚡ TARGET    → What you want to achieve
🟣 ROLE      → Who the AI should be
🟡 REQUEST   → What precisely you want
🔵 OUTPUT    → The format of the response

Step 3: Run Through AI Trigger Protocol

Before executing, verify all 5 triggers are satisfied. The system shows a 🟢 READY state when all checks pass.

Tip: Use the interactive demo to build your first prompt with live validation.

Core Concepts

SIRVATO OS is built on three foundational layers that combine into the Unified Mode:

The Three Layers

  • ICDF Engine — Structural foundation: Instruction, Context, Data, Format
  • MICRO Rhythm — Tonal calibration: Message, Intention, Context, Rhythm, Output
  • RCR-EOC — Complete backbone: Role, Context, Request + Examples, Output, Constraints

The Trigger System

Every prompt flows through the AI Trigger Protocol — a 5-point validation gate that ensures quality before execution. Think of it as automated prompt quality assurance.

AUTO-FLOW Philosophy

SIRVATO OS is designed around progressive disclosure: each completed field triggers the next. This creates a natural, guided flow from intent to execution — the AUTO-FLOW principle.

ICDF Engine Reference

The foundational 4-component engine. Every SIRVATO OS prompt begins here.

Components

  • I — Instruction: Action verb + specific task (e.g., "Write a persuasive email that...")
  • C — Context: Background, audience, scenario, constraints
  • D — Data: Material, facts, input content the AI should use
  • F — Format: Output specification: length, structure, style
# ICDF Template
I → [Action verb] + [specific task]
C → [Scenario], [audience], [background]
D → [Material], [facts], [content]
F → [Format], [length], [style]

MICRO Rhythm Reference

The 5-component layer for tone and emotional calibration.

Components

  • M — Message: Core message or communication focus
  • I — Intention (I²): The desired effect in the reader/user
  • C — Context: Situational nuance and audience awareness
  • R — Rhythm: Tone, energy, pace, emotional texture
  • O — Output: Final delivery form
# MICRO Template
M → [Core message]
I → [Desired effect]
C → [Context + audience]
R → [Tone: confident/urgent/calm] · [Energy: high/medium/low]
O → [Format + length]

Best Practices

Principles for getting the most out of SIRVATO OS.

The more specific your TARGET, the more precise the output. Instead of "write a marketing email," try "write a 200-word B2B cold email for a SaaS project management tool targeting CTOs of 50-200 person companies."
Rhythm is the most under-used component. Define at minimum: tone (professional/casual/urgent), energy (high/medium/low), and perspective (first-person/third-person). This single field transforms generic AI output into crafted communication.
Constraints are not limitations — they're precision tools. Include: what to avoid (no jargon, no passive voice), style rules (Oxford comma, American English), and hard limits (max 150 words, no bullet points).
Provide 1-3 examples in the EXAMPLES field. Counter-examples are especially powerful — showing what you DON'T want calibrates the AI as effectively as positive examples.
Never execute a prompt that hasn't passed all 5 triggers. A prompt that fails Trigger 3 (Data incomplete) will produce hallucinated content. A prompt that fails Trigger 4 (Output ambiguous) will produce something you can't use. The protocol is your quality gate.
