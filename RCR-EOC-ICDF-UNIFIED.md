### Template 2: Comunicazione (RCR-EOC)
```
# SIRVATO — RCR-EOC Console

⚡ LIVE TARGET
Scrivere una email di follow-up post-demo a prospect enterprise.

🟣 ROLE
Senior Account Executive con expertise in SaaS enterprise B2B.

🔵 CONTEXT
Demo avvenuta ieri, 45 minuti. Prospect: CTO di azienda manifatturiera 500 dipendenti.
Interesse alto per funzionalità automazione. Obiezione: integrazione con ERP legacy.

🟡 REQUEST
Email di follow-up professionale che ringrazia, riepiloga i punti chiave della demo
e propone next step concreto (call tecnica entro 1 settimana).

🟣 EXAMPLES
- Stile: professionale ma diretto, non formale
- Lunghezza: 150-200 parole
- Includere: bullet point con 3 punti chiave della demo

🔵 OUTPUT
Email pronta all'invio con oggetto, corpo e firma.

🟡 CONSTRAINTS
- Lingua: italiano
- Evitare: linguaggio di vendita aggressivo
- Includere: proposta valore specifica per il settore manifatturiero
```

### Template 3: Task Complesso (Unified Mode)
```
# SIRVATO OS — Unified Mode

## 1. Role
Architetto software senior specializzato in sistemi distribuiti e cloud-native.

## 2. Instruction
Progetta l'architettura di un sistema di event streaming real-time.

## 3. Request
Diagramma architetturale + ADR (Architecture Decision Record) completo.

## 4. Context
Startup fintech, 2 engineer, budget cloud $2k/mese, GDPR compliance richiesta.
Target: 100k eventi/giorno con latenza < 100ms end-to-end.

## 5. Data
Stack attuale: Node.js, PostgreSQL, AWS. Nessuna esperienza con Kafka.

## 6. Examples
- Pattern: CQRS + Event Sourcing
- Riferimento: architettura Stripe per payment events
- Formato ADR: standard Nygard (Contesto / Decisione / Conseguenze)

## 7. Constraints
- Costo: max $500/mese infrastruttura streaming
- Evitare: over-engineering, managed services costosi
- Priorità: operabilità per team piccolo

## 8. Output
1. Diagramma ASCII dell'architettura
2. ADR in formato Nygard (max 400 parole)
3. Lista tecnologie con giustificazione costo/beneficio
```
